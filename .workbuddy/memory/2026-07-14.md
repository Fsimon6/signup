# 2026-07-14

## 项目扫描分析与凭据安全修复

- 对项目进行了完整扫描，分析了 `sf_automation.py`、`feishu_client.py`、`sf_express.py`、`config.py` 四个核心模块
- 项目架构：从飞书表格读顺丰单号 → Playwright 浏览器自动化查询顺丰 IUOP 运费 → 写回飞书表格
- 发现严重安全问题：`config.py` 中硬编码了飞书 App ID 和 App Secret
- 修复措施：
  - 移除 `config.py` 中所有硬编码凭据，改为仅从环境变量读取
  - 添加 `python-dotenv` 支持（`load_dotenv()`）
  - 创建 `.env.example` 模板文件
  - 创建 `.gitignore` 排除 `.env`、`__pycache__/`、浏览器数据目录
  - 更新 `requirements.txt` 添加 `python-dotenv` 依赖

## sf_click.py 稳定性与金额逻辑改进

- 删除已废弃文件：`debug_feishu.py`、`sf_automation_browser.py`、`feishu_browser.py`
- 修复 `pyautogui.click()` 无坐标问题：改为 `_click_inside_window()` 点击窗口中心内容区
- 修复手动复制金额时读到旧剪贴板内容的问题：手动兜底前先用 `clear_clipboard()` 清空
- 进度文件 `.sf_progress.json` 改为原子写入（先写 .tmp 再替换），避免损坏
- 金额提取逻辑支持「扣款 − 优惠券返款」差额计算（如 92.87 − 1.00 = 91.87）
- 新增 `config.SF_RESULT_WAIT_SECONDS`（默认 3.5s），顺丰结果等待时间可配置
- 剪贴板从 PowerShell `Get-Clipboard` 改为 `pyperclip`（ctypes 直接读写），解决 GBK 编码解码崩溃问题

## sf_click.py 全自动化改造

- 移除所有 `input()` 手动确认交互点：
  - 表格加载：从手动回车改为 `time.sleep(3)` 自动等待
  - SF 输入框聚焦：从手动点击改为 `_click_search_area()` + Ctrl+A 清空
  - 金额提取失败：从手动复制/输入改为自动重试 2 次后跳过
- 标签页编号移入 `config.py`（`FEISHU_TAB_NUMBER` / `SF_TAB_NUMBER`），可通过 `.env` 配置
- 用户只需：1) 登录两个页面；2) 运行脚本，全程无人值守
